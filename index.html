WHY ME?

When I approach programming, I adhere to a set of principles that ensure my code is not only functional but also maintains high standards of quality and efficiency. I encapsulate this philosophy with the acronym SERM, which stands for:

• Scalability

I try my very best to ensure the code that I write is as long-term reliable as possible. The way I do this is by adding flexability to my systems. This allows potential future developers to add onto what's already been written, without any extra effort.

• Efficiency

Alongside scalability, I ensure that my code is as efficient as possible. The way I do this, is by following all the up-to-date best practices. In addition to that, I ensure that my code isn't using any more logic than what it has to. That applies to algorithms as well.

• Reliability

And with efficiency, must come reliability. Efficient code is awesome, but if it's not reliable, all that efficiency goes to waste! Which is why I ensure my code is reliable. Within the framework I use, I ensure that any potential errors are accounted for with my debugging module.

• Modular

What goes well with code that is scalable, efficient and reliable? Modularity! I make my code as modular as possible. Splitting up functions into sub-modules which makes navigating the main module far easier for potential future developers.

Additionally, I love working with other developers to create awesome, innovating things.

Modular Framework Design:

When I first started programming, I found it frustrating to have my content scattered everywhere. To solve this, I began developing a framework that would work for me and any future developers. That's how the first version of the framework came to be, created during a personal project. It wasn't perfect, but it was a solid start. The second version, however, showed much more promise and is displayed on the left.

This version of the framework has many advantages, though it does have a few drawbacks. A significant pro is the networking module. With this module, you can easily access a remote event from anywhere within the .Network folder. Another great feature is how simple it is to develop a base system. All you need to do is create a module within any of the Framework modules and add some initiation code: 

local MyModule = {}

function MyModule.initiate()
    print("Module initiated!")
end

return MyModule

3D Chunk System Design:

Those little cubes might not look very complex, but there's a lot happening behind the scenes. This system was originally designed for multiple use cases, like generating an infinite plane around the player's character or camera.

But that's not all! Since this system is based on Object-Oriented Programming (OOP), we have several methods and functions at our disposal. For example, the :SetAxis(axis, state) method lets you disable a specific axis within the 3D space. In the second video, you'll see that both the 'x' and 'y' axes have been disabled.

You can also disable the chunk system with the :SetActive(state) method.

Number Formatting:

Let's take a look at my number abbreviator/formatter. This small yet complex module is designed to display our base 10 number system in various ways.

The formatter supports multiple types of formatting, including time formats. The video on the left provides a quick showcase of most of these formats.

Pathlink Module:

This neat module creates a path between two points. It's primarily used in tutorials, or guidance on maps.

It supports a variety of customization, from changing the style of each point, to custom marker points along the path.

Loading Screen:

A simple, but effortless way to ensure a proper user experience. This loading screen waits for all content to be loaded before letting the player play. This ensures the player has an optimal experience.

Soft Shutdown:

This soft shutdown module is used to reduce losing players when updating an experience. Upon a server initating a shutdown, the server will delay it slightly and migrate all present players into a reserved server. After a few seconds, all the players will be teleported back into a new, updated server.

Additionally, the user interface is fully customizable and flexible to any necessary changes.
